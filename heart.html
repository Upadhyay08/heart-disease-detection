Exploratory Data Analysis

What the data revealed

Two types of analysis — understanding individual features and their relationship with heart disease.

Numeric Features

  • Age: 40-70 years — both diseased and healthy overlap
  • Cholesterol: right skewed, outliers above 400
  • Oldpeak: heavily right skewed — transformation needed
  • Thalach: diseased patients have HIGHER heart rate (150-180)
  • Trestbps: 120-160 range — outliers present

Top Correlated Features with Target

  • cp (chest pain) → +0.43 — strongest positive signal
  • thalach (max heart rate) → +0.42 — high HR = disease
  • exang (exercise angina) → -0.44 — strongest negative
  • oldpeak (ST depression) → -0.44 — key predictor
  • ca (blocked vessels) → -0.38 — more blocks = less disease?

Key Medical Insight

Higher maximum heart rate (thalach) is associated with heart disease — diseased hearts pump harder. Chest pain type (cp) is the strongest positive predictor. No feature pair has correlation >0.8, so no multicollinearity removal needed.

Feature Types

  • Numeric: age, trestbps, chol, thalach, oldpeak
  • Categorical: sex, cp, fbs, restecg, exang, slope, ca, thal
  • No null values found in any column
  • No multicollinearity — all pairs below 0.8
ML Pipeline

From raw data to 3 models

A reusable pipeline with ColumnTransformer — numeric scaling and categorical encoding in one step.

01

Train / Test Split

80/20 split — 820 train, 205 test. No stratify needed as classes are balanced.

820 / 205
02

ColumnTransformer

Numeric columns → StandardScaler. Categorical columns → OneHotEncoder. Both applied in parallel, fit on train only.

No leakage
03

3 Models Compared

Logistic Regression, KNN (k=5), SVM (rbf kernel) — all in identical pipelines for fair comparison.

Fair comparison
04

Cross Validation (5-fold)

Each model tested on 5 different train/test splits. Average F1 score taken for reliable comparison.

5-fold CV
05

Best Model Selected

SVM won with CV F1 = 0.9329 and lowest variance (±0.023) — most consistent and accurate.

SVM Wins 🏆

Why ColumnTransformer? Different columns need different preprocessing. Numeric features need scaling (StandardScaler), categorical features need encoding (OneHotEncoder). ColumnTransformer applies both in parallel — fit on training data only to prevent data leakage.
